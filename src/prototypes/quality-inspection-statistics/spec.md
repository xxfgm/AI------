# 质量监督抽查统计表 页面规格说明 (Spec)

## 1. 业务功能 (Business Requirements)

### 1.1 核心目标
提供一个“质量监督抽查统计表”的查看页面。基于用户选择的过滤条件（办案机构、查询日期区间、报表周期等），展示指定范围内针对各类产品抽查的企业数、批次数、货值及合格与不合格情况的统计数据。

### 1.2 信息架构 (Information Architecture)
*   **搜索与过滤区 (Query Section)** (与前一页面一致):
    *   **办案机构 (Enforcement Agency)**: 使用基于 `agencyData.ts` 构建的 `Cascader` 级联选择器，支持搜索。
    *   **处罚决定日期 (Decision Date Range)**: 日期范围选择器 (RangePicker)。
    *   **报表周期 (Report Cycle)**: 单选组合 (Radio Group)，包含：月报、一季报、二季报、三季报、年报。
    *   **操作按钮 (Actions)**: 查询 (查询数据) 和 导出 (导出表格)。
*   **数据展示区 (Data Table Section)** (基于新图片):
    *   **指标名称 (Indicator Name)**: 列出不同监控类别。具有层级缩进（例如：“一、劳动防护用品”、“其中：安全帽”等）。
    *   **代码 (Code)**: 对应该项指标的代码。
    *   **抽查企业数 (Number of Inspected Enterprises)**: 包含子标题 `(家)` 和列序号 `1`。
    *   **抽查批次数 (Number of Inspected Batches)**: 包含子标题 `(批次)` 和列序号 `2`。
    *   **货值金额 (Value Amount)**: 包含子标题 `(万元)` 和列序号 `3`。
    *   **合格批次数 (Qualified Batches)**: 包含子标题 `(批次)` 和列序号 `4`。
    *   **不合格批次数 (Unqualified Batches)**: 包含子标题 `(批次)` 和列序号 `5`。
    *   **移送司法机关案件数 (Transferred Cases)**: 包含子标题 `(件)` 和列序号 `6`。

## 2. 布局与结构 (Layout & Structure)
*   **页面整体架构**:
    *   采用单页面滚动架构（Single-page scrolling list）。
    *   背景色为浅灰色 (`bg-gray-50`)。
    *   表单与表格居中显示，最大宽度限制为 `1400px`，确保大屏适配。
    *   页面顶部带有大标题 `质量监督抽查统计表`。
*   **视觉样式 (Visual Style)**:
    *   遵循现有的简约化 Ant Design 自定义风格。
    *   表头：二级和三级嵌套结构，包含主标题、单位（括号内）和序号。
    *   表格行高与斑马线样式与前一页面 (`law-enforcement-statistics`) 保持完全一致。
    *   汇总行（如“合计”）使用加粗文字，数据可点击的下划线/蓝字高亮继续保留。

## 3. 模块详解 (Component Details)

### 3.1 搜索/操作区 (一致性复用)
复用已完成的联动组件逻辑，特别是 `Cascader` 处理机构多级数据。

### 3.2 列表表格组件
*   **字段定义**:
    | 列名 | 字段 Key | 对齐方式 | 渲染逻辑 |
    | :--- | :--- | :--- | :--- |
    | 指标名称 | `indicatorName` | 左对齐 | 顶层加粗；前缀带有“其中：”的内容自动左侧缩进。 |
    | 代码 | `code` | 居中对齐 | 文本展示。 |
    | 抽查企业数 | `enterprises` | 居中对齐 | 表头包含：主标题 -> (家) -> 序号 1。内容使用蓝色链接样式（若有可跳转含义）。 |
    | 抽查批次数 | `batches` | 居中对齐 | 表头包含：主标题 -> (批次) -> 序号 2。 |
    | 货值金额 | `valueAmount` | 居中对齐 | 表头包含：主标题 -> (万元) -> 序号 3。 |
    | 合格批次数 | `qualifiedBatches`| 居中对齐 | 表头包含：主标题 -> (批次) -> 序号 4。 |
    | 不合格批次数 | `unqualifiedBatches`|居中对齐 | 表头包含：主标题 -> (批次) -> 序号 5。 |
    | 移送司法机关案件数| `transferredCases`|居中对齐 | 表头包含：主标题 -> (件) -> 序号 6。 |

*   **Mock 数据要求**: 基于新图片内的数据，提供诸如：合计、劳动防护用品（及安全帽、安全网）、消防器材等分类的假数据（至少 15 条）。

## 4. 交互与响应式 (Interaction)
*   与前一页面相同，支持宽屏下的合理展开，超长表格列宽固定并允许必要时代替折行。
*   级联选择器通过输入法能按拼音或汉字搜索。

## 5. Axure原型支持
*   预留 API 数据请求位置与空状态，目前均使用静态配置模拟原型阶段要求。
