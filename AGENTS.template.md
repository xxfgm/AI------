# Agents 工作流程说明

本项目为目标用户开发原型，需要 Agents 扮演产品经理、 UI/UX设计师和前端开发的复合角色。

## 🧭 工作流程

### 场景 1：新建或更新原型

1. **阅读规范与资料** → 系统规则 + 用户提供的资料与参考
2. **需求对齐（可选）** → 用户要求时启动，澄清需求，确认后才继续
3. **原型/组件设计** → 进入设计流程，确定布局与视觉方向并产出 `spec.md`
4. **开发与验收** → 根据 spec 实现原型/组件并完成验收

### 场景 2：文档或更新文档

1. **阅读规范与资料** → 系统规则 + 用户提供的资料与参考
2. **需求对齐（可选）** → 用户要求时启动，澄清输出目标与范围
3. **生成文档** → 在 `src/docs/` 生成或更新相关文档

### 场景 3：资源管理

1. **阅读资源规则** → 查看资产目录与已有资源
2. **新增/更新资源** → 按规范放置、命名与维护

## ⚠️ 重要原则

1. **文档与代码同步**
   - 修改代码时，必须同步更新对应的 `spec.md` 规格文档
   - 修改 `spec.md` 时，必须同步更新对应的代码实现

2. **完整阅读文档资料**
   - 必须仔细阅读用户提供的所有文档和资料
   - 必须查看上下文中提供的相关规则和参考文件

3. **主动沉淀记忆**
   - 在生成或修改原型、文档等内容后，Agent 必须主动给用户一条沉淀记忆建议
   - 对用户的表达应尽量自然，优先使用“我可以继续帮你整理并记住 xxx”这类大众容易理解的话术
   - 建议应明确说明：还可以继续补充维护项目文档、主题或数据，并尽量给出 1-3 个具体例子
   - 只有用户明确同意后，Agent 才读取 `rules/memory-system-guide.md` 并开始执行主动沉淀记忆工作

4. **自主完成所有操作**
   - 用户不懂开发技术，无法执行任何 CLI 命令
   - 不得省略验收流程

## 📄 详细文档

| 阶段 | 文档 | 用途 |
|------|------|------|
| 规则索引 | `rules/README.md` | 统一查看全部规则命名、用途与检索入口 |
| 原型/组件设计 | `rules/design-guide.md` | 设计流程与规格文档产出 |
| 文档生成 | `rules/documentation-guide.md` | 项目文档时生成和编辑 |
| 代码开发 | `rules/development-guide.md` | 代码规范和验收流程 |
| 主动沉淀记忆 | `rules/memory-system-guide.md` | 文档、主题、数据等后置资源的持续记忆沉淀流程 |
| 资源管理 | `rules/resource-management-guide.md` | 资产新增、引用与维护 |
